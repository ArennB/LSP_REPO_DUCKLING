Heuristic 1:
Name:
Hide implementation details (Information Hiding)
Explanation:
This heuristic states that classes should hide their internal data and implementation details and expose only what is necessary. This improves readability and maintainability because other parts of the program do not depend on internal details that may change. In lecture, this idea was illustrated by limiting access to private variables and avoiding unnecessary public methods.

Heuristic 2:
Name:
Keep methods small and focused on a single responsibility
Explanation:
This heuristic encourages writing methods that perform one clear task instead of multiple unrelated tasks. This improves maintainability because smaller methods are easier to understand, debug, and modify without affecting other parts of the system. In lecture, this concept was explained by breaking large methods into smaller helper methods to improve clarity.

Heuristic 3:
Name:
Minimize coupling between classes
Explanation:
This heuristic states that classes should depend on as few other classes as possible. Lower coupling improves readability and maintainability because changes in one class are less likely to affect others. In lecture, this idea was illustrated by using interfaces or abstract classes to reduce direct dependencies between components.
